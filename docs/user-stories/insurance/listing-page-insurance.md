#### **USER Story:**

**As a user,**
**I want to** view the list of insurance records when I access the Insurance module from the sidebar
**So that** I can easily see and manage all insurance policies created in the system.

---

#### **acceptance criteria:**

1. The user can access the Insurance module from the sidebar of both the Management module and the Chartering module.

2. On clicking, the user should be directed to the Insurance listing page.

3. By default, the listing page is empty and displays records only after insurance records are created.

4. The header of the listing page must include the following fields:

   | **Field** | **Details** |
   | --- | --- |
   | Search Bar | Search field – Placed on the left side of the header section. Searches across Insurance No, Category, Type of Cover, Broker, Insurer / Club, and Linked Entity |
   | Filter | Filter field – Placed after the search bar. Allows filtering by Insurance Category and Insurance Status |
   | Status Group Filters | All, Active, Expired, Cancelled, Closed – Should filter records based on their Insurance Status. **All** is active by default |
   | List / Trash | Group button – By default, the **List** tab should be displayed. Switching to **Trash** shows soft-deleted records |
   | Archived | Toggle – By default, it should be in the **Off** state. When toggled On, only archived insurance records are displayed. Placed after the List / Trash group button |
   | New Insurance | Button – Placed right before the Kebab menu. Opens the Create New Insurance overlay |
   | Kebab Menu | Three-dot icon – Placed at the far right of the header section. The values under this kebab menu are: **1. Delete Selected Insurance  2. Archive Selected Insurance  3. Unarchive Selected Insurance** |
   | Column Show / Hide & Reorder | Show/Hide icon – Placed on the right corner of the column titles row |

5. The following column titles and their dependent values must be displayed on the listing page:

   | **Column Title** | **Condition / Validation** |
   | --- | --- |
   | ID | System-generated sequential number |
   | Insurance No | Auto-generated by the system when creating a record. Following the format: INS-[Current Year]-[Sequential Number] (e.g., INS-2026-001) |
   | Category | The Insurance Category selected during creation (Fixture, Vessel, or Crew) — displayed as a colour-coded badge |
   | Linked Entity | The specific Fixture Number, Vessel Name, or Crew description linked to this record — depends on the selected category |
   | Type of Cover | The type of cover selected during creation, filtered by category |
   | Insurance Type / Clause Type | The clause type selected during creation |
   | Policy Number | The policy number provided during creation |
   | Policy / Cover Reference | The cover reference provided during creation |
   | Currency | The currency selected during creation |
   | Deductible | The deductible value provided during creation, displayed with currency symbol |
   | Sum Insured | The sum insured value provided during creation, displayed with currency (e.g., 10,000,000 USD) |
   | Total Sum Insured (TSI) | The total sum insured value provided during creation, displayed with currency |
   | Annual Premium | The annual premium value provided during creation, displayed with currency |
   | Broker | The broker selected during creation |
   | Broker Reference Number | The broker reference number provided during creation |
   | Broker Contact | The broker contact name or email provided during creation |
   | Insurer / Club | The insurer or P&I club selected during creation |
   | Leading Underwriter | The leading underwriter provided during creation |
   | Insurer Contact | The insurer contact name or email provided during creation |
   | Date of Notification to Broker | The notification date provided during creation |
   | Policy Start Date | The policy start date provided during creation |
   | Policy End Date | The policy end date provided during creation |
   | Insurance Status | The status given during creation or updated in the edit overlay — displayed as a colour-coded badge |
   | Created By | The user selected as the creator during creation |
   | Remarks | The remarks or notes provided during creation |
   | Created Date | The date the record was created, system-generated |

6. **Expandable Linked Entity Records:**

   * Each insurance record is linked to a Fixture, Vessel, or Crew via the category entity field.
   * When a record has a linked entity, an expand arrow is displayed in the first column of that row.
   * Clicking the arrow expands the row to reveal the linked entity as a sub-row directly beneath the parent insurance row.
   * Clicking the arrow again collapses the sub-row.
   * The sub-row must display the following columns:

   | **Sub-list Column** | **Details** |
   | --- | --- |
   | Item | The identifier of the linked entity (e.g., AEW-F-2026-407, MV OCEAN STAR) |
   | Type | The category of the linked entity: Fixture, Vessel, or Crew — displayed as a colour-coded badge |
   | Name | The display name of the linked entity |

   * Sub-rows are visually indented and styled distinctly from the main record row (e.g., lighter background).
   * Multiple rows can be expanded simultaneously.

7. Since the number of columns exceeds screen limitations, **horizontal scrolling** must be implemented to allow users to navigate and view all details.

8. The functionality for Search and Filter, New Insurance, Kebab menu, and Column Show/Hide & Reorder has been covered in separate user stories.

9. All insurance records must be sorted based on their **created date and time** — latest created first.

10. The **column width adjustment** functionality must be available on the listing page, allowing users to resize columns as needed.

11. **List / Trash tab behaviour:**
    * **List** (default): Displays all active (non-deleted) insurance records.
    * **Trash**: Displays only soft-deleted records, with options to restore or permanently delete them.

12. **Archived toggle behaviour:**
    * When the Archived toggle is **Off** (default): Only non-archived records are shown.
    * When the Archived toggle is **On**: Only archived records are shown in the listing.
    * The toggle state is independent of the List / Trash tab selection.

13. A **footer** must be displayed at the bottom of the listing page showing:
    * Total number of records found (e.g., *Found 12 records*)
    * Items per page selector (25 / 50 / 100)
    * Current range and total count (e.g., *1 – 25 of 47*)
    * Previous / Next page navigation buttons
