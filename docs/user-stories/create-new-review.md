#### **USER Story:**

**As a user,**
**I want to** create a new legal review by entering relevant review details with guided field dependencies
**So that** legal reviews are recorded accurately, consistently, and linked to the correct records.

#### **acceptance criteria:**

1. A Create New Review overlay should be available for the user when clicking on **"Add Review"** from the Legal Review listing page.

2. The Overlay should have the following functionalities:
   1. | **Field Name** | **Values** | **Validations** | **Visibility Condition** | **Comments / Dependencies** |
      | --- | --- | --- | --- | --- |
      | Review No | Auto-generated by the system when creating a review. Following the format: REV-[Current Year]-[Sequential Number] (e.g., REV-2026-0001). | Read-only | Always visible | System-assigned; not editable by user |
      | Related To | Searchable multi-select field | Optional | Always visible | User can search and select any Inquiry, Fixture, or Scope record by ID or name. Multiple records can be selected. Helper text shown: *"Select Inquiry / Fixture / Scope"* |
      | Review Type | Documents, NDA | Mandatory | Always visible | Enables Save button when filled |
      | Description | Free text | Optional, Max 10,000 chars | Always visible | Character counter displayed |
      | Legal Review Status | To be reviewed by Legal, Under Review, Reviewed, Not Required | Mandatory | Always visible | Enables Save button when filled |
      | E Filing Number | Alphanumeric free text | Optional, Max 50 chars | Always visible | Non-alphanumeric characters are silently stripped on input |
      | Due Date | Date picker | Optional | Always visible | No date constraints |
      | Review Raised By | Dropdown (user list) | Optional | Always visible | Single select from system user list |
      | To Be Reviewed By | Dropdown (user list) | Optional | Always visible | Single select from system user list |

   ### **Related To Field Behaviour**

   * The **Related To** field is a single searchable input — there is no separate "Items" field.
   * Typing in the field filters across all Inquiry, Fixture, and Scope records simultaneously.
   * Matching results are shown in a dropdown grouped by record type (Inquiry / Fixture / Scope) with their IDs and names.
   * The user can select multiple records; each selection appears as a removable chip above the search input, colour-coded by record type:
     * Inquiry — blue
     * Fixture — violet
     * Scope — orange
   * A chip can be removed by clicking the × on it.
   * Searching by category keyword (e.g. "Inquiry") shows all records of that type.

   ### **Action Fields**

   | **Field Name** | **Values** | **Validations** | **Visibility Condition** | **Comments / Dependencies** |
   | --- | --- | --- | --- | --- |
   | Create Review | Button | Enabled only when Review Type AND Legal Review Status are both filled | Always visible | Submits the form |
   | Cancel | Button | No validation | Always visible | Closes the overlay without saving |

3. **Button Enable Logic:**
   * The **Create Review** button must remain disabled until the following mandatory fields are filled:
     * Review Type
     * Legal Review Status
   * All other fields are optional and do not block submission.

4. Once the user clicks **Create Review**, the review is created immediately with no confirmation pop-up and:
   * A sequential Review No is assigned (e.g., REV-2026-0001)
   * `Created Date` is set to the current date
   * The new record appears at the **top of the Legal Review listing**, sorted by latest created
   * The overlay closes automatically

5. Clicking **Cancel** closes the overlay without saving any data. No confirmation is required.

6. The **Task Management** section is available only in **Edit mode** (after a review has been created), not during creation.

7. Whenever a Review is created and linked to an Inquiry, Fixture, or Scope via the Related To field:
   * The respective record should reflect this linkage in its own detail view.

8. Whenever a linked Inquiry, Fixture, or Scope record is removed or deleted from the system:
   * The field values in the Review edit overlay must **not** be automatically removed.
   * A validation warning must be shown:
     * *"The following linked records are no longer available: [list of affected records]"*
   * This validation must be treated as a **warning** — the user can proceed without updating the fields.
