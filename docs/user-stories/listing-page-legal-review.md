#### **USER Story:**

**As a user,**
**I want to** view the list of reviews when I access the Legal Review module from the sidebar
**So that** I can easily see and manage all legal reviews created in the system.

#### **acceptance criteria:**

1. The user can access the Legal Review module from the sidebar.

2. On clicking, the user should be directed to the Legal Review listing page.

3. By default, the listing page is empty and displays records only after reviews are created.

4. The header of the listing page must include the following fields:

   | **Field** | **Details** |
   | --- | --- |
   | Search Bar | Search field – Placed on the left side of the header section |
   | Add Filter | Filter button – Placed after the search bar |
   | List / Trash | Group button – By default, the **List** tab is active. Switching to **Trash** shows soft-deleted reviews |
   | Archived | Toggle – By default, it must be in the **Off** state. When toggled On, only archived reviews are displayed |
   | Add Review | Button – Placed right before the Kebab menu. Opens the Create New Review overlay |
   | Kebab Menu | Three-dot icon – Placed at the far right of the header section. Contains: **1. Delete Selected Review  2. Archive Selected Reviews  3. Unarchive Selected Reviews** |
   | Column Show / Hide & Reorder | Show/Hide icon – Placed on the right corner of the column titles row |

5. The following column titles and their dependent values must be displayed on the listing page:

   | **Column Title** | **Condition / Validation** |
   | --- | --- |
   | Expand / Collapse | An expand arrow (▶) is shown when a review has one or more linked records in the Related To field. Clicking it expands the row to show linked records as sub-rows. If no linked records exist, the column is empty for that row |
   | ID | System-generated sequential number |
   | Review No | Auto-generated by the system when creating a review. Following the format: REV-[Current Year]-[Sequential Number] (e.g., REV-2026-0001) |
   | Review Type | The review type selected during creation (Documents or NDA) |
   | Description | The description provided during creation |
   | Legal Review Status | The status given during creation or updated in the edit overlay |
   | E Filing Number | The alphanumeric E Filing number provided during creation |
   | Due Date | The due date provided during creation |
   | Review Raised By | The user selected as the reviewer who raised the review |
   | To Be Reviewed By | The user selected to review |
   | Created Date | The date the review was created, system-generated |

6. **Expandable Related Records:**

   * Each review may be linked to one or more Inquiry, Fixture, or Scope records via the **Related To** field.
   * When a review has at least one linked record, an expand arrow (▶) is displayed in the first column of that row.
   * Clicking the arrow expands the row to reveal a sub-list of all linked records directly beneath the parent review row.
   * Clicking the arrow again (now ▼) collapses the sub-list.
   * The sub-list must display the following column titles:

   | **Sub-list Column** | **Details** |
   | --- | --- |
   | Item | The ID of the linked record (e.g., CR-2026-272, AEW-F-2026-407) |
   | Type | The category of the linked record: Inquiry, Fixture, or Scope — displayed as a colour-coded badge |
   | Name | The display name of the linked record |

   * Each linked record is displayed as its own sub-row under the parent review.
   * Sub-rows are visually indented and styled distinctly from the main review row (e.g., lighter background).
   * Multiple reviews can be expanded simultaneously.

7. Since the number of columns exceeds screen limitations, **horizontal scrolling** must be implemented to allow users to navigate and view all details.

8. The functionality for Search and Add Filter, Add Review, Kebab menu, and Column Show/Hide & Reorder has been covered in separate user stories.

9. All reviews must be sorted based on their **created date and time** — latest created first.

10. The **column width adjustment** functionality must be available on the listing page, allowing users to resize columns as needed.

11. **List / Trash tab behaviour:**
    * **List** (default): Displays all active (non-deleted) reviews.
    * **Trash**: Displays only soft-deleted reviews, with options to restore or permanently delete them.

12. **Archived toggle behaviour:**
    * When the Archived toggle is **Off** (default): Only non-archived reviews are shown.
    * When the Archived toggle is **On**: Only archived reviews are shown in the listing.
    * The toggle state is independent of the List / Trash tab selection.

13. A **footer** must be displayed at the bottom of the listing page showing:
    * Total number of records found (e.g., *Found 12 records*)
    * Items per page selector (25 / 50 / 100)
    * Current range and total count (e.g., *1 – 25 of 47*)
    * Previous / Next page navigation buttons
